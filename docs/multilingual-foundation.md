# Multilingual Foundation

## Що вже закладено

- У `courses` та `lessons` додаються поля:
  - `learning_language_code`
  - `translation_language_code`
- API `GET /learning/courses` і `GET /learning/courses/:courseId/lessons` тепер приймають optional query:
  - `learningLanguage`
  - `translationLanguage`
- Frontend зберігає вибір мови навчання в `localStorage` через `platform/frontend/learning-language.js`.

## Поточна стратегія

- Поточний контент лишається англомовним (`en`) з українською підтримкою (`ua`).
- Італійська (`it`) додається як окрема мова навчання на рівні курсів/уроків, а не як "режим перекладу" поверх тих самих уроків.
- Прогрес не потрібно мігрувати окремо по мові, поки англійські та італійські уроки мають різні `lesson_id`.

## Наступний етап

1. Додати італійські курси та уроки з `learning_language_code = 'it'`.
2. Розширити контент-модель слів/фраз:
   - або окремими `it_text` полями як короткий шлях,
   - або окремою таблицею локалізацій як правильний довгий шлях.
3. Оновити тренажерний flow так, щоб перевірка відповідей і підказки працювали для `it`.

## Чому не додаємо італійську "поверх" англійської

- Поточна контент-модель жорстко орієнтована на `en_text` та `ua_text`.
- Якщо швидко "доліпити" `it`, логіка стане трикратно заплутанішою.
- Поточна основа дозволяє додавати окремі італійські курси поступово, не ламаючи англійську версію.
